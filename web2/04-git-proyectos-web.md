# Control de versiones con Git en proyectos web

El desarrollo frontend actual no se reduce a escribir componentes o estilos. También exige ordenar cambios, recuperar versiones previas, trabajar con historial y sostener una práctica de trabajo que facilite la colaboración y el mantenimiento. Por ese motivo, Git ocupa un lugar importante en el recorrido de la materia.

Este capítulo introduce las operaciones esenciales de control de versiones desde una perspectiva práctica y vinculada con proyectos web.

## Objetivos del capítulo

- comprender la utilidad del control de versiones;
- reconocer la diferencia entre directorio de trabajo, staging e historial;
- utilizar operaciones básicas para registrar cambios;
- integrar Git como parte del flujo normal de trabajo.

## Contenidos mínimos

### Qué problema resuelve Git

Se presenta Git como una herramienta para registrar cambios, volver a estados previos y mantener trazabilidad sobre la evolución del proyecto.

### Repositorios, commits y ramas

Se introducen los conceptos mínimos que permiten leer y construir un historial de trabajo comprensible.

### Flujo de trabajo básico

Se trabajan operaciones como inicializar un repositorio, agregar archivos, confirmar cambios y revisar el historial.

### Buenas prácticas iniciales

Se proponen criterios simples para nombrar commits, evitar desorden y sostener una rutina de versionado coherente.

## Relación con el proyecto integrador

Desde las primeras versiones del e-commerce conviene registrar avances parciales, separar cambios y sostener un historial claro que ayude a revisar decisiones posteriores.

## Cierre

Después del control de versiones, el siguiente paso consiste en comprender el entorno de trabajo que hoy acompaña a casi todos los proyectos frontend: Node.js y npm.
