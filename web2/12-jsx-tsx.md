# JSX y TSX: sintaxis declarativa para interfaces

Una de las primeras particularidades que aparecen al trabajar con React es la presencia de una sintaxis que combina estructura de interfaz y expresiones de JavaScript dentro de un mismo archivo. Comprender esa sintaxis es importante porque organiza la lectura cotidiana de componentes y constituye la base inmediata del trabajo práctico.

Este capítulo presenta JSX y TSX como recursos de escritura declarativa para describir interfaces de manera clara y cercana a la estructura visual que luego se renderiza.

## Objetivos del capítulo

- comprender la función de JSX y TSX en React;
- distinguir sintaxis declarativa de HTML tradicional;
- reconocer expresiones, atributos y composición dentro de componentes;
- preparar la escritura de componentes propios.

## Contenidos mínimos

### Qué es JSX

Se explica JSX como una sintaxis que permite describir la interfaz usando una mezcla controlada de estructura y expresiones de JavaScript.

### Qué cambia en TSX

Se introduce TSX como la variante que integra JSX con TypeScript, permitiendo tipar props, eventos y estructuras de datos.

### Expresiones y atributos

Se revisa el uso de llaves, atributos, clases y diferencias frecuentes con HTML tradicional.

### Lectura de componentes reales

Se propone una primera aproximación a componentes escritos con JSX y TSX para mejorar la comprensión del formato.

## Relación con el proyecto integrador

Las vistas principales del e-commerce comenzarán a describirse mediante JSX y, progresivamente, mediante TSX en componentes tipados.

## Cierre

Comprendida la sintaxis, ya es posible avanzar hacia la unidad de trabajo más importante de React: el componente.
