# Web Components como estándar de la plataforma

El enfoque basado en componentes no pertenece exclusivamente a una biblioteca determinada. La propia plataforma web fue incorporando mecanismos para encapsular estructura, comportamiento y estilos en piezas reutilizables. Por ese motivo, conviene detenerse brevemente en Web Components como respuesta nativa del ecosistema.

## Objetivos del capítulo

- reconocer a Web Components como estándar de la plataforma;
- identificar sus ideas principales de encapsulamiento y reutilización;
- ubicar su relación con otras herramientas del ecosistema;
- ampliar el marco conceptual del desarrollo por componentes.

## Contenidos mínimos

### Estándares y componentes nativos

Se presenta la idea de que la web ofrece mecanismos propios para definir elementos reutilizables sin depender de un framework específico.

### Custom elements y encapsulamiento

Se introducen de manera general nociones como elementos personalizados y aislamiento de responsabilidades.

### Ventajas y límites

Se discute qué problemas resuelven y en qué situaciones pueden resultar menos convenientes que otras herramientas.

### Lugar de Web Components en el ecosistema actual

Se ubica este enfoque como parte del panorama general del frontend, no como reemplazo obligatorio de React.

## Relación con el proyecto integrador

Aunque el proyecto principal se desarrollará con React, este capítulo aporta contexto para entender que la noción de componente tiene raíces más amplias en la propia plataforma web.
