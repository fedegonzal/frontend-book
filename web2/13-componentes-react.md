# Componentes en React

Si React organiza la interfaz a partir de componentes, conviene detenerse en qué significa construirlos, cómo se definen y qué papel cumplen dentro de una aplicación. En esta etapa ya no alcanza con comprender la idea general de componente: hace falta empezar a usarla como herramienta de diseño y composición.

## Objetivos del capítulo

- definir componentes funcionales en React;
- comprender cómo se componen entre sí;
- reconocer criterios básicos de división de interfaz;
- preparar la introducción de props y estado.

## Contenidos mínimos

### Componentes como unidades de trabajo

Se presenta el componente como una unidad reutilizable que encapsula estructura, lógica y, en algunos casos, comportamiento de una parte de la interfaz.

### Componentes funcionales

Se introduce el enfoque actual basado en funciones como forma principal de definir componentes en React.

### Composición de componentes

Se trabaja la construcción de interfaces complejas a partir de la combinación de componentes más pequeños.

### Criterios iniciales de división

Se proponen preguntas básicas para decidir cuándo conviene separar una pieza de interfaz en un componente propio.

## Relación con el proyecto integrador

Las pantallas del e-commerce comenzarán a organizarse como una composición de tarjetas, listados, secciones de detalle, navegación y formularios.

## Cierre

Una vez definidos los componentes, el siguiente paso consiste en entender cómo intercambian información mediante props y cómo gestionan cambios internos con state.
