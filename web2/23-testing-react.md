# Testing de interfaces con Jest y React Testing Library

Construir una interfaz que parece funcionar no siempre alcanza. En proyectos reales también importa verificar comportamientos, reducir regresiones y sostener cierta confianza sobre el sistema cuando el código cambia. El testing cumple ese papel y permite pasar de una validación puramente manual a una práctica más sistemática.

Este capítulo introduce Jest y React Testing Library como herramientas iniciales para el trabajo con pruebas de interfaz.

## Objetivos del capítulo

- comprender por qué el testing forma parte del desarrollo frontend;
- introducir pruebas de componentes con Jest y React Testing Library;
- reconocer qué comportamientos conviene verificar;
- preparar el cierre integrador del proyecto.

## Contenidos mínimos

### Qué significa testear una interfaz

Se presenta el testing como una práctica de validación que acompaña el desarrollo y no solo como control posterior.

### Jest como entorno de pruebas

Se introduce Jest de manera general como herramienta para ejecutar y organizar pruebas automatizadas.

### React Testing Library

Se presenta esta biblioteca como una forma de probar componentes desde la perspectiva de su comportamiento visible.

### Casos iniciales de prueba

Se proponen escenarios sencillos de validación sobre renderizado, interacción y presencia de contenido.

## Relación con el proyecto integrador

Las vistas principales del e-commerce y algunos comportamientos clave podrán validarse mediante pruebas básicas orientadas a la interfaz.

## Cierre

Con el testing como parte del proceso, el recorrido puede concentrarse en integrar todas las piezas del proyecto en una aplicación más coherente.
