Qué es Grid
CSS Grid Layout es un sistema de diseño en dos dimensiones que permite crear estructuras de filas y columnas de manera muy flexible.

NPM es un gestor de paquetes para Javascript (técnicamente para Node.js) y es una de las herramientas más utilizadas en el desarrollo web moderno. 

Permite compartir, instalar y administrar librerías y herramientas de JavaScript de manera sencilla y ordenada, pensada para equipos diversos y también para desarrolladores individuales.

NPM no se instala por separado, viene incluido con Node.js. 

1. Descargar Node.js desde la página oficial https://nodejs.org 
2. Elegir la versión recomendada (LTS = Long Term Support) para mayor estabilidad

Una vez instalado, abrir la terminal o consola y comprobar con:
node -v 		→ muestra la versión de Node.js
npm -v 		→ muestra la versión de NPM
NPM (node package manager)

NPM suele usarse para cuatro funciones:

1. Instalar dependencias (librerías)
2. Gestionar versiones (por versión o por sistema operativo)
3. Automatizar tareas mediante scripts definidos en el archivo package.json
4. Publicar y compartir paquetes con la comunidad global

Application Programming Interface (API)

El concepto de API es muy amplio y abarca cuestiones funcionales, estructurales, tecnológicas y hasta documentales. Por ejemplo, los lenguajes de programación (como Javascript, Python, etc) tienen palabras clave que usamos para programar (let, while, function, etc) esas palabras clave son "la API" del lenguaje para que nosotros escribamos código que el lenguaje pueda comprender.

En esta materia nos interesan las APIs de datos y servicios. Son las que usaremos para que nuestro frontend interactúe con información que reside en otros sistemas (por ejemplo, una base de datos en un servidor)
Distintos tipos de APIs

REST (Representational State Transfer)
- El más común hoy en día.
- Usa HTTP y formatos como JSON.

GraphQL
- Similar a REST, pero permite pedir
  exactamente los datos que necesitan.
- Usa HTTP y formatos como JSON.
Distintos tipos de arquitecturas APIs
SOAP (Simple Object Access Protocol)
- Más antiguo, usa XML, es más rígido.


WebSockets
Permiten una conexión bidireccional y persistente entre cliente y servidor. Normalmente devuelve JSON. Útiles para tiempo real.

Representational State Transfer (REST) es un estilo de arquitectura para diseñar APIs basadas en HTTP. La arquitectura define cómo se interactúa con HTTP. Si bien las APIs REST no definen el formato de intercambio de información, lo más común es usar JSON para enviar y recibir los datos.

Muchas veces suele verse el concepto de RESTFul. Esto no es más que otra forma de decir "REST". En la práctica, REST es la definición y RESTFul su correcta implementación. En la materia hablaremos de REST haciendo referencia a ambos.
Qué es una API REST

Cliente - Servidor: separación entre cliente (app) y servidor (API).

Sin estado (stateless): no hay sesión, cada petición envía y recibe todo lo necesario.

Cacheable: las respuestas pueden almacenarse en caché para optimizar.

Interfaz uniforme: recursos se identifican con URLs (ejemplo: /users/1).

Sistema en capas: puede haber proxys, balanceadores, etc. sin afectar al cliente.
Principios de una API REST

Cada petición a una API REST va acompañada de la acción que se quiere ejecutar sobre un recurso (o endpoint), esto habitualmente se llama método o verbo HTTP.

Por ejemplo, para solicitar información sobre un recurso, se utiliza el verbo GET.

GET /users		→ devuelve una lista con datos de todos los usuarios
GET /users/15	→ devuelve datos del usuario 15
Acciones posibles sobre una API REST

Acciones posibles sobre una API REST

Algunas APIs ofrecen la posibilidad de agregar parámetros opcionales en la URL del recurso a solicitar. Esto es útil por ejemplo para pedir que los datos devueltos estén ordenados o filtrados bajo cierto criterio, etc. Es muy útil para paginar resultados.

La disponibilidad de estos parámetros opcionales dependerá de la implementación de cada API, no es una función automática.

GET /productos?categoria=verduleria&atributos=organico,local

Web Components

Web Components es un conjunto de tecnologías que permite crear elementos personalizados reutilizables, con su funcionalidad aislada del resto del código. Es un estándar oficial mantenido por W3C e implementado en navegadores modernos.

A diferencia de React (que es una librería), Web Components es nativo en la web.

https://github.com/WICG/webcomponents 
https://developer.mozilla.org/en-US/docs/Web/API/Web_components 
Un estandar de la W3C, nativo en browsers

Reutilizables: mismo componente en múltiples páginas.

Fáciles de mantener: separación clara de responsabilidades.

Soporte nativo en navegadores: sin dependencias de frameworks o librerías.

Mejora progresiva: funciona con HTML/CSS/JS básicos.
Ventajas de Web Components

Es una API de JavaScript para crear etiquetas personalizadas y definir su comportamiento. Luego pueden usarse en la interfaz como si fueran etiquetas HTML normales. Las etiquetas se crean con JS y se usan como si fueran HTML.

<mi-componente> </mi-componente>

https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_custom_elements 
Custom Elements

Asocia un árbol DOM encapsulado a un elemento. Permite mantener estilo y funcionalidad privados. Evita colisiones con otros elementos del documento.

Esta ventaja de encapsulación es muy importante, pero también puede hacer que el contenido del componente sea más difícil de acceder y modificar desde el exterior, algo que a veces es necesario.

https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_shadow_DOM
Shadow DOM

Custom Elements es la API para definir una etiqueta nueva y darle comportamiento propio. Es indispensable para crear Web Components.

Shadow DOM es la API para encapsular el estilo y la estructura de un componente, evitando colisiones con el resto del documento. Es opcional, aunque se recomienda su uso para una mayor modularidad.
Creación de un componente:

class ProductoItem extends HTMLElement {
    constructor() {
        super();
    }
}

customElements.define(
'producto-item', ProductoItem
)
Resumen de conceptos

connectedCallback() se ejecuta al insertar un elemento en el DOM.
render() se encarga de actualizar el contenido del componente.

disconnectCallback() se ejecuta al eliminar un elemento del DOM.
renderError() se ejecuta si hay un error en render().

El componente debe registrarse y asignarle un nombre. El nombre debe llevar guión.
customElements.define('producto-item', ProductoItem)

Métodos básicos

Los Web Components pueden recibir parámetros (atributos), también tienen propiedades.

<producto-item titulo="Remera mangas cortas blanca"></producto-item> 

se leen dentro del componente con this.getAttribute('titulo')
Si un atributo cambia en tiempo de ejecución, se puede utilizar el método attributeChangedCallback() para reaccionar al cambio.

Hablamos de propiedades en vez de atributos cuando los utilizamos en JavaScript: miElemento.titulo = "Remera mangas cortas negra";
Definición de parámetros

Un componente puede importar y usar otros. Esto se llama composición y permite crear interfaces complejas desde componentes simples.

Por ejemplo, un componente <productos-list> puede importar y utilizar internamente múltiples <producto-item>, encargándose cada uno de mostrar la información de un producto. De esta forma se logra una estructura modular, donde cada pieza tiene una responsabilidad clara, y al combinarlas se obtiene una interfaz escalable y fácil de mantener, siguiendo el principio de “divide y vencerás”.

Lit es una librería de Google para crear web components de manera sencilla, rápida y eficiente. Introducción a React

El corazón de React son los componentes, piezas independientes que representan partes de la interfaz, como botones o formularios. Al combinarlos, se construyen aplicaciones completas.

Un componente puede definirse como una función que devuelve JSX, una sintaxis similar a HTML.
Componentes en React

Conceptos básicos en React
Props, la comunicación entre componentes: permiten transmitir información de un componente padre a uno hijo. Funcionan como parámetros de funciones y hacen que los componentes sean reutilizables y flexibles.

State, datos internos del componente: representa variables internas que cambian con el tiempo, como un contador o el valor de un input. Al modificarse, React vuelve a renderizar automáticamente el componente (propiedad "reactiva").

Los Hooks son funciones especiales que permiten a los componentes funcionales usar estado y otras capacidades avanzadas de React. Son la base del desarrollo moderno en esta biblioteca.

React maneja eventos como clicks o cambios en inputs con una sintaxis propia y un sistema de eventos virtual que optimiza el rendimiento respecto a JavaScript puro.

Los formularios se controlan vinculando el valor de los inputs al estado. Esto permite validaciones, sincronización y un control más preciso de los datos.
Conceptos básicos en React

La composición permite crear componentes más complejos combinando otros más simples. React fomenta esta práctica en lugar de la herencia.

Lifting State Up: cuando varios componentes necesitan acceder al mismo dato, el estado se eleva a un componente padre, que lo distribuye mediante props. Este patrón mantiene la lógica centralizada.

React Router permite construir SPA, gestionando rutas sin recargar la página y ofreciendo una experiencia fluida al usuario.

React Router

Un router es un sistema que permite controlar la navegación entre diferentes vistas o páginas dentro de una aplicación. En el contexto del desarrollo web, el router se encarga de interpretar la URL que el usuario visita y decidir qué componente debe cargarse. En las aplicaciones de una sola página (SPA, Single Page Application), el router evita que se recargue toda la página al cambiar de vista, logrando una navegación más rápida y fluida.
Introducción a React Router

React Router es una librería oficial de React que permite implementar un sistema de rutas dentro de una aplicación de una sola página. Gracias a ella, los desarrolladores pueden definir qué componentes se deben renderizar según la URL actual, sin recargar el sitio completo. 

React Router ofrece herramientas como <Routes>, <Route>, <Link>, useNavigate() y useParams() para manejar la navegación, los enlaces, los parámetros y las transiciones entre vistas de forma declarativa y eficiente.
Introducción a React Router

Existen distintos tipos de rutas. Las estáticas no reciben ningún parámetro (por ejemplo, /inicio o /contacto). 

Las dinámicas cambian según los valores que se pasen a través de la URL (por ejemplo, /producto/:id) donde :id es un parámetro posicional (obligatorio) que indica qué elemento se debe mostrar. 

También existen los parámetros nombrados (opcionales), conocidos como parámetros de consulta o query, que se pasan después del signo ? en la URL (por ejemplo, /productos?categoria=ropa&orden=asc). Estos permiten enviar información adicional de forma más flexible, sin alterar la estructura principal de la ruta.

Diferencia entre params y query params

Params → forman parte de la ruta (/usuarios/42)

Query params → se pasan en la URL con ? (/usuarios/42?destacar=true)

Para leer query params, se usa el hook useSearchParams():
import { useSearchParams } from "react-router-dom";

function BuscarUsuario() {
 const { id } = useParams();
 const [searchParams] = useSearchParams();
 const destacado = searchParams.get("destacar");

 return 
   <p>Usuario {id} estará destacado {destacado}</p>;
}
