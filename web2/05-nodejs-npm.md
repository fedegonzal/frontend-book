# Node.js y npm como entorno de trabajo

Aunque el frontend se ejecuta en el navegador, el desarrollo cotidiano suele apoyarse en herramientas que funcionan fuera de él. Node.js y npm forman parte de ese entorno y permiten instalar dependencias, ejecutar scripts y organizar proyectos modernos con mayor previsibilidad.

Este capítulo introduce esos conceptos como base práctica para el resto del recorrido, especialmente para trabajar con React y otras bibliotecas del ecosistema.

## Objetivos del capítulo

- comprender el rol de Node.js en el desarrollo frontend;
- reconocer a npm como gestor de paquetes y scripts;
- interpretar la función del archivo `package.json`;
- preparar el entorno para proyectos con herramientas modernas.

## Contenidos mínimos

### Qué es Node.js

Se presenta Node.js como entorno de ejecución de JavaScript fuera del navegador y como base de muchas herramientas de desarrollo.

### Qué es npm

Se introduce npm como gestor de dependencias y scripts, indispensable para trabajar con proyectos contemporáneos.

### Package.json y dependencias

Se explica la función del archivo `package.json`, la diferencia entre dependencias y scripts, y su papel en la organización del proyecto.

### Rutinas de trabajo básicas

Se plantean prácticas mínimas para instalar paquetes, ejecutar scripts y mantener un entorno de desarrollo consistente.

## Relación con el proyecto integrador

La aplicación e-commerce utilizará dependencias y scripts para iniciar el proyecto, incorporar bibliotecas y sostener un flujo de trabajo estable.

## Cierre

Con los fundamentos del entorno ya establecidos, el recorrido puede pasar a una pregunta más conceptual: por qué el desarrollo de interfaces se organiza hoy en torno a componentes.
