# Arquitectura de frontend y organización del código

Cuando una aplicación crece, ya no alcanza con que cada componente funcione de manera aislada. También importa cómo se organiza el proyecto, dónde se ubican los archivos, cómo se separan responsabilidades y qué criterios permiten sostener el código con el paso del tiempo.

Este capítulo introduce un conjunto de decisiones arquitectónicas básicas para que el trabajo con React no quede reducido a una suma de archivos dispersos.

## Objetivos del capítulo

- reconocer la importancia de una arquitectura de frontend clara;
- organizar carpetas y archivos con criterios comprensibles;
- distinguir componentes presentacionales y contenedores;
- promover separación de responsabilidades desde etapas tempranas.

## Contenidos mínimos

### Organización de carpetas

Se presentan criterios iniciales para distribuir vistas, componentes, servicios, estados y recursos del proyecto.

### Componentes presentacionales y contenedores

Se introduce esta distinción como herramienta para ordenar responsabilidades y evitar componentes excesivamente cargados.

### Separación de responsabilidades

Se trabaja la conveniencia de desacoplar interfaz, lógica de negocio, acceso a datos y estado.

### Escalabilidad y mantenimiento

Se relaciona la organización del código con la facilidad para leer, modificar y extender la aplicación.

## Relación con el proyecto integrador

La arquitectura definida en este punto servirá como base para distribuir vistas, componentes, hooks, store y servicios del e-commerce.

## Cierre

Con esta base conceptual y organizativa, el libro puede ingresar en su núcleo aplicado: el desarrollo web con React.
