# Introducción a TypeScript para frontend

En proyectos pequeños puede parecer suficiente escribir solo JavaScript, pero a medida que una interfaz crece aparecen estructuras de datos más complejas, más funciones y más posibilidades de error. En ese contexto, TypeScript ofrece una ayuda importante para documentar supuestos y detectar inconsistencias antes de ejecutar el código.

Este capítulo presenta una introducción gradual a TypeScript, pensada como apoyo para el resto del libro y especialmente para su incorporación progresiva en ejemplos con TSX durante la parte dedicada a React.

## Objetivos del capítulo

- comprender qué problema intenta resolver TypeScript;
- reconocer la noción de tipado estático gradual;
- escribir anotaciones básicas de tipos en variables y funciones;
- preparar la lectura de componentes escritos con TSX.

## Contenidos mínimos

### Qué es TypeScript

Se presenta TypeScript como una capa de tipos sobre JavaScript que agrega verificación y mejora la comunicación entre personas, herramientas y código.

### Tipos primitivos y estructuras simples

Se introducen tipos básicos, arreglos, objetos y uniones simples como punto de partida para describir datos.

### Tipado de funciones

Se revisa cómo tipar parámetros y valores de retorno, con énfasis en claridad y legibilidad.

### TypeScript en proyectos de frontend

Se adelanta el papel de TypeScript en componentes, props, respuestas de API y formularios.

## Relación con el proyecto integrador

La tipificación progresiva permitirá describir productos, respuestas de API, props y estructuras de estado con mayor precisión a medida que el proyecto avance.

## Cierre

Con este marco ya es posible complementar la escritura de código con una práctica necesaria en proyectos reales: el control de versiones mediante Git.
