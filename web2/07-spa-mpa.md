# SPA y MPA: modelos de navegación y composición

No todas las aplicaciones web organizan la navegación de la misma manera. Algunas se estructuran como un conjunto de páginas que el navegador solicita por separado, mientras que otras resuelven gran parte de la navegación dentro de una única aplicación cargada inicialmente. Comprender esta diferencia es importante para ubicar mejor el lugar que ocupan React y otras herramientas del frontend actual.

## Objetivos del capítulo

- distinguir aplicaciones MPA y SPA;
- reconocer diferencias de navegación y composición;
- interpretar ventajas y límites de cada enfoque;
- ubicar a React dentro del modelo de página única.

## Contenidos mínimos

### Qué es una MPA

Se presenta el modelo de múltiples páginas como enfoque tradicional de la web, basado en solicitudes completas al servidor ante cada navegación.

### Qué es una SPA

Se introduce la aplicación de página única como modelo donde gran parte de la navegación ocurre sin recarga completa del documento.

### Comparación entre ambos enfoques

Se discuten diferencias en experiencia de usuario, complejidad técnica, organización del código y manejo de datos.

### Implicancias para el desarrollo frontend

Se relaciona la elección del modelo con el uso de ruteo, estado local y consumo de APIs.

## Relación con el proyecto integrador

La aplicación e-commerce se pensará como una SPA, lo que permitirá organizar vistas, navegación y estado dentro de un mismo proyecto frontend.

## Cierre

Antes de concentrarse en React, conviene revisar qué ofrece la propia plataforma web para trabajar con componentes mediante estándares nativos.
