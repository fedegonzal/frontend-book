# Introducción a React y su modelo de trabajo

React ocupa un lugar central en el desarrollo frontend contemporáneo porque ofrece una forma de construir interfaces basada en componentes, composición y actualización declarativa. Antes de usar sus herramientas concretas conviene comprender qué problemas resuelve y por qué su modelo de trabajo se volvió tan difundido.

Este capítulo abre la parte principal del libro y presenta el marco general desde el cual se desarrollarán los capítulos siguientes.

## Objetivos del capítulo

- ubicar a React dentro del ecosistema frontend actual;
- comprender su lógica declarativa y basada en componentes;
- reconocer su relación con estado, interfaz y renderizado;
- preparar el recorrido práctico de la parte 3.

## Contenidos mínimos

### Qué es React

Se presenta React como biblioteca orientada a la construcción de interfaces a partir de componentes reutilizables.

### Pensamiento declarativo

Se introduce la idea de describir qué interfaz debería mostrarse según el estado de la aplicación, en lugar de manipular manualmente cada detalle del DOM.

### Componentes y composición

Se retoma la noción de componente ahora dentro del contexto específico de React.

### React en el flujo del proyecto

Se ubica su papel dentro del desarrollo del e-commerce y dentro del ecosistema de herramientas que acompañan a una aplicación real.

## Relación con el proyecto integrador

Desde este punto el proyecto transversal comienza a construirse efectivamente con React como tecnología principal de interfaz.
