# Librerías de componentes y criterios de adopción

En muchos proyectos no se construye toda la interfaz desde cero. Existen bibliotecas de componentes que ofrecen piezas reutilizables, sistemas visuales y patrones ya resueltos. Sin embargo, usarlas con criterio exige evaluar ventajas, límites y costos de integración.

Este capítulo propone una introducción a ese problema desde una perspectiva formativa y no meramente instrumental.

## Objetivos del capítulo

- reconocer qué aportan las librerías de componentes;
- discutir criterios de adopción en proyectos reales;
- relacionar velocidad de desarrollo con consistencia y dependencia externa;
- preparar decisiones de integración en el proyecto final.

## Contenidos mínimos

### Qué es una librería de componentes

Se presenta este tipo de biblioteca como un conjunto de piezas visuales reutilizables y, en muchos casos, coherentes con un sistema de diseño.

### Ventajas de adopción

Se discuten beneficios posibles como velocidad, consistencia y reducción de trabajo repetitivo.

### Límites y costos

Se analizan cuestiones como personalización, dependencia externa, peso conceptual y adaptación a necesidades reales.

### Criterios para elegir

Se proponen preguntas orientadoras para decidir si conviene o no incorporar una biblioteca de este tipo.

## Relación con el proyecto integrador

El e-commerce podrá incorporar algunos componentes externos de manera controlada, siempre que esa decisión resulte coherente con el objetivo pedagógico de la materia.

## Cierre

Antes del cierre final del libro, conviene introducir una práctica clave del desarrollo profesional: el testing de interfaces.
