# Renderizado condicional y listas

Gran parte de las interfaces reales depende de condiciones y colecciones de datos. Una pantalla puede mostrar un mensaje distinto si no hay resultados, una lista de productos si la consulta fue exitosa o una advertencia si ocurrió un error. Por ese motivo, renderizar según condiciones y recorrer listas forma parte del trabajo cotidiano en React.

## Objetivos del capítulo

- renderizar interfaz en función de condiciones simples;
- representar colecciones de datos mediante listas;
- reconocer la importancia de las claves en elementos repetidos;
- preparar el trabajo con datos remotos y estados de interfaz.

## Contenidos mínimos

### Condiciones en la interfaz

Se presentan formas comunes de mostrar u ocultar partes de la interfaz según el estado de la aplicación.

### Renderizado de listas

Se trabaja la representación de arreglos de datos dentro de la interfaz declarativa.

### Claves y estabilidad

Se introduce la necesidad de usar claves adecuadas para que React identifique correctamente los elementos listados.

### Escenarios frecuentes

Se conectan condiciones y listas con casos habituales como loading, ausencia de datos o conjuntos filtrados.

## Relación con el proyecto integrador

El listado de productos, los mensajes de error, el carrito vacío o las vistas filtradas del e-commerce dependen directamente de estos patrones.
