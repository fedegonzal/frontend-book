# Organización del libro

Este libro es la bibliografía de apoyo de la materia Desarrollo Web 2, correspondiente a la Tecnicatura Universitaria en Desarrollo de Aplicaciones de la UNTDF. Fue pensado para estudiantes que ya recorrieron una primera etapa de desarrollo web y necesitan avanzar hacia herramientas, arquitecturas y prácticas más cercanas al trabajo actual de frontend.

El recorrido se organiza en cuatro partes. La primera reúne fundamentos del desarrollo web moderno: JavaScript actual, asincronía, consumo de APIs, TypeScript, Git y Node.js con npm. La segunda trabaja el enfoque basado en componentes y ofrece un puente conceptual entre la web como plataforma y los marcos de trabajo modernos. La tercera desarrolla React como herramienta principal para construir interfaces. La cuarta retoma testing, integración, despliegue y cierre del recorrido.

A lo largo del libro se propone un proyecto integrador transversal: una aplicación e-commerce que se construye de manera incremental sobre una API real documentada con Swagger. Ese proyecto permite articular contenidos conceptuales con decisiones de implementación, organización y validación.

Al finalizar este libro, se espera que el lector pueda:

- comprender el papel de JavaScript moderno, TypeScript y Node.js dentro del ecosistema frontend;
- consumir APIs REST y organizar el trabajo asincrónico en aplicaciones de interfaz;
- reconocer el valor del enfoque basado en componentes y de una arquitectura de frontend mantenible;
- construir interfaces con React usando componentes, props, state, hooks, rutas y estado global;
- integrar formularios, consumo de datos y librerías de componentes con mayor criterio técnico;
- incorporar prácticas iniciales de testing, integración y despliegue en un proyecto de frontend.

## Cómo usar este libro

Este material fue pensado para acompañar una cursada intermedia de frontend. No busca reemplazar la documentación oficial de cada herramienta ni funcionar como una referencia exhaustiva de todo el ecosistema, sino ordenar un recorrido progresivo que ayude a estudiar, practicar y relacionar conceptos.

Conviene recorrer el libro en orden. La primera parte establece bases técnicas que luego se reutilizan en el resto del trayecto. La segunda parte ofrece el marco conceptual necesario para entender por qué React y otras bibliotecas trabajan con componentes. La tercera parte concentra la mayor parte del desarrollo aplicado, y la cuarta parte permite cerrar el recorrido con una mirada de integración más completa.

Siempre que sea posible, se recomienda trabajar con el editor abierto y reproducir ejemplos, pequeñas variaciones y ejercicios. En esta etapa de la formación ya no alcanza con leer definiciones: conviene observar cómo se conectan herramientas, cómo se estructura el código y qué decisiones simplifican o complejizan un proyecto real.

## Herramientas mínimas para empezar

Para aprovechar este libro conviene contar con las siguientes herramientas:

- un navegador moderno con herramientas de desarrollo;
- un editor de código, preferentemente Visual Studio Code;
- Node.js y npm instalados para gestionar dependencias y ejecutar proyectos;
- Git para versionar el trabajo realizado a lo largo de la cursada.

También resulta útil disponer de una carpeta de trabajo ordenada y de una rutina técnica básica: crear repositorios, nombrar archivos con criterio, separar componentes y recursos, revisar errores en consola y registrar cambios de manera frecuente.

## Estructura de la materia

El libro sigue la misma lógica general de la materia Desarrollo Web 2. Primero se consolidan herramientas y conceptos del ecosistema actual. Luego se introduce el enfoque basado en componentes y se analiza su valor arquitectónico. Más adelante se trabaja React como herramienta principal de construcción de interfaces. Finalmente se incorporan testing, integración y criterios iniciales de despliegue.

Como hilo conductor práctico, el recorrido retoma el desarrollo de una aplicación e-commerce. El proyecto permite trabajar listado de productos, detalle, carrito, formularios de checkout, estado compartido, consumo de API y validación mediante pruebas.

Con este marco, el siguiente capítulo se ocupa de JavaScript moderno como base para comprender la mayor parte de las herramientas y bibliotecas que aparecen en el frontend actual.
