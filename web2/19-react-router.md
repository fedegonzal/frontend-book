# Navegación en aplicaciones con React Router

Las aplicaciones de página única no delegan toda la navegación al servidor. Por ese motivo, necesitan herramientas que permitan organizar vistas, rutas y transiciones dentro del propio frontend. React Router cumple ese papel en gran parte del ecosistema React y por eso ocupa un lugar importante en el recorrido del libro.

## Objetivos del capítulo

- comprender por qué una SPA necesita un sistema de ruteo;
- introducir React Router como herramienta de navegación;
- relacionar rutas con componentes y vistas;
- preparar la estructura de pantallas del proyecto integrador.

## Contenidos mínimos

### Rutas en aplicaciones React

Se presenta la noción de ruta como mecanismo para asociar segmentos de navegación con vistas o componentes.

### Configuración básica de React Router

Se introduce la idea general de definir rutas, enlaces y componentes asociados.

### Parámetros y navegación

Se adelanta el uso de parámetros para resolver vistas como el detalle de un producto.

### Organización de vistas

Se relaciona el sistema de rutas con una estructura de proyecto más clara y mantenible.

## Relación con el proyecto integrador

El e-commerce requerirá al menos una vista de listado, una vista de detalle y una navegación hacia carrito o checkout.

## Cierre

Con la navegación resuelta, el siguiente paso consiste en estudiar cómo manejar estado compartido de una manera más explícita mediante Zustand.
